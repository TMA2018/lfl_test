Что имеем?
----------
1) HTML-разметку для отрисовки матч-постов для соц.сетей:
https://sheets.amateum.com/graphic/611a13fe643d22548b08a89c

2) Шаблон с настройками привязанный к конкретной лиге:
GET https://rest.amateum.com/templates/611a138d728e210c2c38d795

3) Поле customStyle в структуре шаблона со строкой стилей, которые "переезжают" стили по-умолчанию
"customStyle":"#sharable {background: linear-gradient(to top, #1F1C18, #8E0E00);}.sharable-stripes-left {background: linear-gradient(to top, #1F1C18, #331913);}.sharable-cont{background: rgba(0, 0, 0, .75);}.tournament .inner .emblem-text, .sharable-stripes-right > div:first-child {color: #401710;}.sharable-stripes-right > div:last-child {background: #8a2210;}.scores > div {background-color: rgba(105, 23, 10, .35);}.events-item:nth-child(even) {background-color: rgba(138, 34, 16, .15);}"}

----------

Что надо сделать?
----------
Простой UI на реакте, где в левой части экрана будет iframe с картинкой (по адресу разметки),
а в правой части - окошко редактора кода. В этом окошке выводится и редактируется содержимое поле customStyle

Ожидаемый результат:
-----------
1) для редактора кода используется любая на выбор библиотека из NPM, с оговоркой, что она сможет распарсить строку стилей в человекочитаемый "блочный" вид
2) при изменении кода стилей должен отправляться запрос на сохранение шаблона методом PUT на url по которому шаблон изначально получен.
3) у нас REST-апи, поэтому тело PUT-запроса состоит из одного поля: {customStyle: "наша_новая_строка_стилей"}
4) чтобы запрос не отправлялся после изменения каждого символа кода, используем метод debounce из библиотеки throttle-debounce с порогом в 5000 мс
5) после получения 200 ответа от сервера (шаблон сохранён), нужно сделать reload нашему iframe, чтобы отобразились изменения в картинке
